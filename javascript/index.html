JS-Basics

JavaScript uses variables to "catch and hold values". After a variable has been defined, such as
var ten = 5 * 2
ten can then be used to refer to that variable. The variable can be redefined with an =, or operated on by all the built in Math operators.

Arithmetic in JavaScript is pretty standard. If you do not specify the order of operations then JavaScript will take care of it for you. It knows that
100 + 4 * 11 = 144
The modulo operator is % and it returns the remainder
var moduwhat? = 5 % 2 = 1
The modulo operator has the same precedence as multiplication and division.

Strings are used to represent text. The origin supposedly stems from the fact that strings hold characters together. (H-A-P-P-Y B-I-R-T-H-D-A-Y). There are a few special things to know about strings. For example, whenver a backslash (\) is used it indicates that the following character has a special meaning.
console.log("This is the first line\nThis is the second line")
=> This is the first line
=> This is the second line

+ operators can be used to concatenate strings as well.
"hello" + " " + "world"
=> "hello world"

There are certain keywords and reserved words that can't be used as variable names. Watch out for char and class keywords especially:
abstract boolean break byte case catch char class const continue debugger default delete do double else enum export extends false final finally float for function goto if implements import in instanceof int interface long native new null package private protected public return short static super switch synchronized this throw throws transient true try typeof var void volatile while with

This is an example of a string variable. Concantenation can be used to add on to the string, the string itself can be changed to "Susan", however the string cannot be interpolated. The variable must be separated out from the string like so:
console.log("Hello" + " " + name)
=> "Hello Phil"

This is an example of a number. Plain and simple.
console.log(num + " is the square root of " + num*num)
=> 4 is the square root of 16

An array is an object that is formatted as a collection of things. Usually these things are similar. One of the most useful asepcts of an array is its built-in length property. Calling .length on an array returns the number of items in the array. The length property will be useful later on when we start talking about for loops. Array values can be accessed through the index, which starts at 0.
array[2]
=> "bob"

This is where things start to get interesting. In JavaScript objects are used to store properties. Most value types have fixed properties, meaning they can't be changed, you just use them and access them. For example, string has a property .length that counts the characters in a given string. That string has a fixed length that never changes. However, when you create your own objects you can give them whatever properties you want, remove them, and change them at your will. The main role of objects is to be a collection of properties. More on objects in the next section.

Objects & Loops

As we saw before, objects are used to store properties. Here, a:1 is a property, with a as a key and 1 as the value of a.

Object get really interesting when we start nesting other objects inside of them. Functions can also be nested inside objects.

There are a few ways to access properties from an object. You can use dot notation or bracket notation.

object.b
=> 2
object["b"]
=> 2

Adding a property to an object in JavaScript is easy. You can use bracket notation or dot notation to do so. You can also set the value to undefined.
object.c = 3
console.log(object)
=> { a:1, b:2, c:3 }

You can also create a new object using a constructor function, which is similar to initializing the instance of a class in Ruby.

function Car(make, model, year) {
this.make = make;
this.model = model;
this.year = year;
}

var myCar = new Car("Eagle", "Talon TSi", 1993)
console.log(myCar.make)
=> "Eagle"

Use either Object.keys(object) or Object.getOwnPropertyNames(object) to access the keys from an object and return them in the form of an array.
Object.keys(object)
=> ['a', 'b', 'c']

The for...in loop iterates over the properties of an object in an arbitrary order. If a property is modified in one iteration and then visited a later time, it becomes the value from which it was last modified.

var o = {a:1, b:2, c:3}
function show_properties(object) {
for(var property in object) {
console.log(object[property])
}
}

show_properties(o)
=> 1
=> 2
=> 3

For loops are great for accessing elements inside of an arrray. Unlike Ruby, JavaScript does not come with built-in counter methods like .each or .map. So you're going to have to build your own. Here's an example of a for loop console.logging an array's elements.
array = ['you', 'me', 'he', 'she']
for(var i=0; i<array.length; i++) {
console.log(array[i])
}
=> you
=> me
=> he
=> she
